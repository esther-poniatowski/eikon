# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-02-11

### Added

- Declarative YAML-based figure specification with `FigureSpec` and `PanelSpec`.
- 4-layer configuration hierarchy: defaults, `eikon.yaml`, figure overrides, runtime kwargs.
- Style system with 3 built-in presets (publication, presentation, poster), inheritance via `extends`, and `.mplstyle` support.
- Layout engine with gridspec-based placement, shared axes, colorbars, and insets.
- 13 built-in plot types (8 matplotlib, 5 seaborn) via `@plot_type` decorator.
- Export pipeline with PDF, SVG, PNG, and TIFF handlers; metadata injection for PDF and PNG.
- Figure registry with YAML persistence, tag/group filtering, and conflict handling.
- CLI commands: `init`, `validate`, `render`, `batch`, and registry subcommands (`list`, `add`, `remove`, `show`).
- Plugin discovery via entry points with `register_plot_type`, `register_transform`, and `register_hook`.
- Margin labels for annotating rows/columns with hierarchical, per-label styling.
- `FigureHandle` return type with `.save()`, `.show()`, `.close()`, and `.path(fmt)` methods.
- `register_format_handler()` for custom export format handlers, extending the plugin pattern.
- `parse_layout_spec()` in `spec._parse` for parsing layout dicts independently.
- `merge_spec_override()` now supports replacing `panels` (was silently skipped).
- Multi-panel example spec (`multi-panel-example.yaml`) generated by `eikon init`.
- `examples/` directory with 5 runnable demo scripts.
- `sphinx.ext.autodoc` + `sphinx.ext.napoleon` configured for auto-generated API docs.
- Comprehensive type annotations passing `mypy --strict`.
- 497 tests with ~94% coverage.
- Sphinx documentation: 10 user guide pages + 9 API reference pages.

### Fixed

- `inject_pdf_metadata` no longer destroys the PDF (was truncating via `PdfPages`; now uses `pypdf`).
- `render_registry.py` Path truthiness: `Path("")` fallback to `specs_dir` now triggers correctly.
- `_placement_cells` handles zero-valued slice attributes (`slice(0, 0)` no longer expands).
- `ResolvedPaths` exported in `__all__` (was used in public `render()` signature but not exported).
- `FigureHandle.save()` raises `RuntimeError` when `figure` is `None`; `close()` is a safe no-op.
- Silent `except Exception` in `render()` now emits `warnings.warn()` so config errors surface.
- `_parse_mplstyle` strips inline comments from values (e.g. `12 # points` no longer stores the comment).
- `title_kwargs`, `shared_legend`, and `hide_spines` are now parsed from YAML specs.
- Per-panel `style` overrides are now applied during the draw pass.
- User styles can `extends` other user styles (not only built-in presets).
- `_parse_layout_spec()` moved from `render._pipeline` to `spec._parse` for proper separation of concerns.
